Cortex-R7 Startup Example Code for Arm Compiler for Embedded 6 - Arm®Development Studio

This example shows the bare-metal startup code for the Cortex-R7 processor, including vector table, exception handlers, MPU, cache, TCM and FPU initialization, and is illustrated by a simple semihosted "sorts" example application.

Purpose and scope

This example shows the bare-metal startup code for the Cortex-R7 processor, including vector table, exception handlers, MPU, cache, TCM and FPU initialization. A semihosted "sorts" example is provided in sorts.c for demonstration purposes, that can be replaced by your own application code. This example can be run on either the Cortex-R7 FVP model or your own target hardware. This example can be used as a framework on which to build your own C/C++ applications.

Hardware and software requirements

To run this example on your own target hardware, you will need:
  • Your Cortex-R7 target hardware and a suitable power supply for it
  • DSTREAM or ULINK debug hardware and a suitable power supply for it, and a JTAG cable to connect it to your Cortex-R7 target hardware
  • A platform configuration for your Cortex-R7 target hardware, created with the Platform Configuration Editor (PCE)
A ready-made launch configuration startup_Cortex-R7x1-FVP.launch (to connect to the FVP model) is provided.

Building the example

This example is intended to be built with Arm Compiler for Embedded 6 using the supplied Eclipse project, or directly on the command-line with the supplied makefile. If you wish to modify and rebuild the example, you must use Arm Compiler for Embedded 6 to rebuild it.
The Eclipse project is a managed builder project, that creates a makefile in the /Debug directory.
The application executable is built to be loaded and run in RAM at address 0x0, but could be easily ported to platforms with RAM at other addresses by simply changing the code and data addresses in the scatter-file scatter.scat.
The assembler startup code is assembled into Arm instructions and the C application code is compiled into Thumb instructions (with the --thumb switch).
This example depends on semihosting support being provided by the debug system. The Debugger enables semihosting automatically if either symbols __auto_semihosting or __semihosting_library_function are present in an image. Arm Compiler for Embedded 6 and Arm Compiler 5 both add __semihosting_library_function automatically to an image at link time if that image uses any semihosting-using functions. If compiling with GCC, use an alias symbol definition such as void __auto_semihosting(void) __attribute__ ((alias("main"))); or turn on semihosting support in the Debugger explicitly with set semihosting enabled on.

Building on the command-line

To build the example on the command-line with the supplied make utility:
  • On Windows, open an Arm DS Command Prompt from the Start menu, run the select_toolchain utility, and select Arm Compiler for Embedded 6 from the list
  • On Linux, run the suite_exec utility with the --toolchain option to select the compiler and start a shell configured for the Development Studio environment, for example: ~/developmentstudio/bin/suite_exec --toolchain "Arm Compiler for Embedded 6" bash
Then navigate to the ...\startup_Cortex-R7_AC6 directory, and type:
make
The usual make rules: clean, all and rebuild are provided in the makefile.

Building within the IDE

To build the supplied projects within the IDE:
  1. In the Project Explorer view, select the project you want to build.
  2. Select Project → Build Project.

Running the example on Cortex-R7x1 FVP model

  1. Select Run → Debug Configurations....
  2. Select the startup_Cortex-R7x1-FVP from the list of Generic Arm C/C++ Application configurations.
  3. Click on Debug to start debugging. The example executable will be downloaded to the target, and the program counter PC will be set to the entry point of the image.
  4. Run the executable (press F8). Text output appears in the Target Console view, similar to:
Cortex-R7 bare-metal startup example
Floating point calculation using the FPU...
Float result is        0.937500
Float result should be 0.937500
Insertion sort took 2 clock ticks
Shell sort took 0 clock ticks
Quick sort took 1 clock ticks
You can also run the example executable directly on the FVP model from within the Arm DS Command Prompt on Windows or the suite_exec shell on Linux with, for example:
FVP_VE_Cortex-R7x1 -a startup_Cortex-R7_AC6.axf

Running the example on your Cortex-R7 target hardware

  1. Power-up your Cortex-R7 target hardware
  2. Select Run → Debug Configurations....
  3. Select the platform configuration you created with PCE from the list of platforms.
  4. In the Connections panel, enter the USB: or TCP: IP address or name of your DSTREAM or ULINK unit in the Debug Hardware Address field, or click on Browse to select one from a list, otherwise an error will be reported: Launch configuration has errors: Configuration for connection type 'Bare Metal Debug' is not valid - Connection cannot be empty .
  5. Click on Debug to start debugging. The example image will be downloaded to the target and program execution will be held at Start.
  6. Run the executable (press F8). Text output appears in the App Console view, similar to:
Cortex-R7 bare-metal startup example
Floating point calculation using the FPU...
Float result is        0.937500
Float result should be 0.937500
Insertion sort took 2 clock ticks
Shell sort took 0 clock ticks
Quick sort took 1 clock ticks

Debugging the reset handler

To debug the reset handler, first disconnect any existing debug session, then power-cycle your Cortex-R7 target hardware, then load the executable (startup_Cortex-R7_AC6.axf) in the same way as before, selecting "Debug from entry point" in the Debugger tab.
The Debugger will download the program's code and data sections to the target, and set the program counter PC to the entry point of the image at Start, at the reset entry in the vector table in startup.s.
  1. In the Registers view, expand Core to see the core registers.
  2. Execute the LDR PC, Reset_Addr instruction by single-stepping (press F5) into Reset_Handler. Notice the Program Counter (PC) change in the Core registers view.
  3. The first instructions inside Reset_Handler perform a read/modify/write (MRC, BIC, MCR) of CP15 Control Register to disable the MPU, branch prediction and caches. In the Registers view, expand CP15, expand Control, then expand SCTLR. Single-step (press F5) through these instructions, and see the M, C, Z and I bits being cleared (if they were previously set).
  4. The next instructions initialize the Supervisor mode stack. Expand Core in Registers view. Single-step (press F5) through these instructions and see SP being initialized to address 0x10000 as determined by the scatter-file scatter.scat.
  5. The next instructions (MOV, MCR) invalidate the Instruction and Data caches. Single-step (press F5) through these instructions.
  6. The next block illustrates basic TCM configuration, as the basis for exploration by the user. These are conditionally assembled via the TCM switch, which is not active by default.
  7. The next block configures the MPU for 4 regions (startup code, application code, data, stack/heap) to addresses determined by the scatter-file scatter.scat. In the Registers view, expand CP15, then expand MPU. Single-step (press F5) through these instructions, and see the region number, region base and region access control being set for each region.
  8. The next block enables access to the FPU and switches on the FPU hardware, if present. This is conditionally assembled in only when assembling for Cortex-R7 with an FPU.
  9. Finally, the MPU is enabled and code execution branches to __main to setup the C library run-time environment.
  10. Delete all breakpoints, and continue running (press F8) until the application terminates normally.
