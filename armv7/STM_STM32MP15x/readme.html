STM (System Trace Macrocell) and Event Viewer Example for Arm Compiler for Embedded 6 and ST STM32MP15x platform - Arm®Development Studio

This example illustrates the use of STM (System Trace Macrocell) on the ST STM32MP15x platform and the Debugger's Event Viewer.

Purpose and scope

This example illustrates the use of STM (System Trace Macrocell) on the ST STM32MP15x platform and the Debugger's Event Viewer, based on a modified version of the Primes example. This makes use of two STM channels to transport text and data with timestamps. A ready-made launch configuration STM_STM32MP15x.launch is provided. This example is derived from the blog Programming ARMs System Trace Macrocell.

Hardware and software requirements

  • ST STM32MP15x Evaluation platform and a suitable power supply for it
  • DSTREAM-family debug hardware unit with its power supply, and JTAG and/or Mictor cable to connect to the ST STM32MP15x platform
  • Alternatively, a ULINKpro (D) or ULINK2 debug unit can also be used
The source code is intended to be built with Arm Compiler for Embedded 6 and depends on semihosting support being provided by the debug system. Arm Compiler for Embedded 6 is provided with Development Studio. The Debugger will automatically enable semihosting for the image when it is built with Arm Compiler for Embedded 6.

Building within the IDE

To build the supplied projects within the IDE:
  1. In the Project Explorer view, select the project you want to build.
  2. Select Project → Build Project.

Building on the command-line

To build the example on the command-line with the supplied make utility:
  • On Windows, open an Arm DS Command Prompt from the Start menu, run the select_toolchain utility, and select Arm Compiler for Embedded 6 from the list
  • On Linux, run the suite_exec utility with the --toolchain option to select the compiler and start a shell configured for the Development Studio environment, for example: ~/developmentstudio/bin/suite_exec --toolchain "Arm Compiler for Embedded 6" bash
Then navigate to the ...\STM_STM32MP15x directory, and type:
make
The usual make rules: clean, all and rebuild are provided in the makefile.

Running the example

  1. Connect the DSTREAM/ULINK to the target by the JTAG cable to CN14 and Mictor cable to U10 (see below).
  2. If using a ULINK(pro) device then just connect the JTAG cable to CN14
  3. Power-up the DSTREAM
  4. Ensure STM32MP15x is set to boot from SDCard (SW1[1..3] set to ON-OFF-ON) and suitable Linux image is present on SDCard.
  5. Let STM32MP15x's boot loader initialize, but interrupt UEFI/U-Boot before booting Linux.
  6. Select Run → Debug Configurations....
  7. Select the STM_STM32MP15x from the list of Generic Arm C/C++ Application configurations.
  8. For DSTREAM units, in the Connections panel, enter the USB: or TCP: IP address or name of your DSTREAM unit in the Debug Hardware Address field, or click on Browse to select one from a list, otherwise an error will be reported: Launch configuration has errors: Configuration for connection type 'Bare Metal Debug' is not valid - Connection cannot be empty .
  9. For ULINK units, in the Connections panel, click on Browse to select unit from a list, otherwise an error will be reported: Launch configuration has errors: Configuration for connection type 'Bare Metal Debug' is not valid - Connection cannot be empty .
  10. Click on the "DTSL Options" Edit... button.
  11. In the Trace Buffer tab, select the Trace capture method as either On Chip Trace Buffer (CSTMC/ETF) (for a JTAG cable connection) or DSTREAM 4GB Trace Buffer (for a Mictor cable connection to DSTREAM) or Streaming Trace (for a Mictor to JTAG+CS20 connection to DSTREAM-ST/-PT).
  12. In the STM tab, tick Enable STM Trace. Click on OK to save the DTSL options.
  13. Click on Debug to start debugging.
  14. The example image will be downloaded to the target and run to main().
  15. In the Events view, open the View Menu, then open the Event Settings. Ensure the Trace Source matches the Trace capture method specified earlier, e.g. ETF:STM then press Reset to select Binary decoding for all Ports/Channels 0-65535.
  16. In the Events view menu bar, click on Clear Trace, then run the executable (press F8).
  17. After the executable has finished executing, in the Events view menu bar, click on Decode events from the beginning of the trace buffer.
